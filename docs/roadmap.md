# DMARRSS Roadmap

Here is a high-level project roadmap with milestones and estimated timelines:

1. **Data Collection and Preprocessing** (1-2 weeks)

- [ ] [1.1](https://github.com/orgs/folkvarlabs/projects/5?pane=issue&itemId=23920355). Finalize a list of relevant cybersecurity datasets, logs, and threat intelligence feeds.

- [ ] [1.2](https://github.com/orgs/folkvarlabs/projects/5?pane=issue&itemId=23920355). Complete the development of preprocessing scripts to clean, format, and normalize the data.

- [ ] [1.3](https://github.com/orgs/folkvarlabs/projects/5?pane=issue&itemId=23920355). Successfully split the preprocessed data into training, validation, and test sets.

2. **Model Development and Fine-tuning** (3-4 weeks)

- [ ] [2.1](https://github.com/orgs/folkvarlabs/projects/5?pane=issue&itemId=23920359). Choose the appropriate LLM-inspired architecture and neural network technologies for the system.

- [ ] [2.2](https://github.com/orgs/folkvarlabs/projects/5?pane=issue&itemId=23920359). Complete the implementation and fine-tuning of the models on the preprocessed cybersecurity datasets.

- [ ] [2.3](https://github.com/orgs/folkvarlabs/projects/5?pane=issue&itemId=23920359). Successfully incorporate context-aware and risk-sensitive mechanisms into the model.

3. **System Integration and Evaluation** (2-3 weeks)

- [ ] [3.1](https://github.com/orgs/folkvarlabs/projects/5?pane=issue&itemId=23920362). Develop scripts for combining Context Aware Event Severity Layers and Numerically Augmented Data Source, and verify their effectiveness.

- [ ] [3.2](https://github.com/orgs/folkvarlabs/projects/5?pane=issue&itemId=23920362). Complete the integration of the DMARRSS system with existing security tools like SNORT and SURICATA/ZEEK.

- [ ] [3.3](https://github.com/orgs/folkvarlabs/projects/5?pane=issue&itemId=23920362). Evaluate the system's performance using standard metrics and confirm satisfactory results.

4. **Visualization and Frontend Development** (2-3 weeks)

- [ ] [4.1](https://github.com/orgs/folkvarlabs/projects/5/views/1?pane=issue&itemId=23921066). Design a user-friendly frontend interface for visualizing the finalized threat event table.

- [ ] [4.2](https://github.com/orgs/folkvarlabs/projects/5/views/1?pane=issue&itemId=23921066). Implement the frontend with support for different log formats and customization options.

- [ ] [4.3](https://github.com/orgs/folkvarlabs/projects/5/views/1?pane=issue&itemId=23921066). Test the frontend to ensure seamless integration with the DMARRSS system and reliable performance.

5. **Testing and Validation** (1-2 weeks)

- [ ] [5.1](https://github.com/orgs/folkvarlabs/projects/5?pane=issue&itemId=23920379). Complete comprehensive testing of the system, including unit tests, integration tests, and end-to-end tests.

- [ ] [5.2](https://github.com/orgs/folkvarlabs/projects/5?pane=issue&itemId=23920379). Validate the system's accuracy and effectiveness in detecting and prioritizing threat events.

- [ ] [5.3](https://github.com/orgs/folkvarlabs/projects/5?pane=issue&itemId=23920379). Address any identified issues and confirm that the system meets all requirements.

6. **Documentation and Deployment** (1-2 weeks)

- [ ] [6.1](https://github.com/orgs/folkvarlabs/projects/5/views/1?pane=issue&itemId=23920379). Prepare comprehensive documentation covering system architecture, usage, and troubleshooting.

- [ ] [6.2](https://github.com/orgs/folkvarlabs/projects/5/views/1?pane=issue&itemId=23920379). Package the system for deployment, ensuring compatibility with the target infrastructure.

- [ ] [6.3](https://github.com/orgs/folkvarlabs/projects/5/views/1?pane=issue&itemId=23920379). Successfully deploy the DMARRSS system and confirm smooth operation in the target environment.

7. **Maintenance and Updates** (Ongoing)

- [ ] [7.1](https://github.com/orgs/folkvarlabs/projects/5/views/1?pane=issue&itemId=23920381). Establish a monitoring process to continuously track the system's performance and address any issues that arise.

- [ ] [7.2](https://github.com/orgs/folkvarlabs/projects/5/views/1?pane=issue&itemId=23920381). Periodically update the system with new data, models, and improvements to maintain its effectiveness.

- [ ] [7.3](https://github.com/orgs/folkvarlabs/projects/5/views/1?pane=issue&itemId=23920381). Implement a feedback loop to incorporate user insights and enhance the system's capabilities over time.
